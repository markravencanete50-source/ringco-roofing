Google reviews

Read and reply to your Google Business Profile reviews without leaving this dashboard.

One-time Google connection

  1. In Google Cloud Console, create a project and enable the Business Profile APIs — Google requires a short access request (free, usually approved in a few days).
  2. Create an OAuth client ID (type: Web application) and add this site's URL to Authorized JavaScript origins.
  3. Paste the client ID into GOOGLE_CLIENT_ID at the top of /js/admin.js and redeploy.

Once connected, every review appears below with a reply box — replies publish straight to your public Google listing.

Finish setup

You're signed in, but Firebase isn't connected yet — so there's no live data to show. Two quick steps switch this into your live leads dashboard:

  1. Connect Firebase. The web config is read automatically at deploy from your Vercel NEXT_PUBLIC_FIREBASE_* environment variables — nothing to paste. If you're seeing this on the live site, confirm those six vars exist in Vercel → Settings → Environment Variables.
  2. Turn on email/password login & publish the rules. Firebase → Authentication → Sign-in method → enable Email/Password. Then Authentication → Users → Add user with your admin email and a password. Make sure a /users doc exists with that same email, role: "admin", status: "active", and publish the rules from /firestore.rules.

Once connected, every request from your website contact form lands here automatically — nothing else to set up. Optional: to also embed Google Analytics traffic charts, add a GA4 ID to /js/analytics.js and a Looker embed URL to /js/admin.js.
